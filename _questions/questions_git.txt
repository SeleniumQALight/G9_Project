

   ** Git **
        1. послідовність дій для створення бранчі
            1.1 перевірити чи є зміни в local changes, якщо є то зробити:
                - commit
                - rollback
                - stash
            1.2 переключитися в main
            1.3 зробити pull з origin/main
            1.4 опціонально перевірити чи main є еталонним
                (робимо push і дивимося що не має коммітів і НЕ пушимо)
            1.5 створюємо нову гілку

        2. послідовність дій для пуша змін
            2.1 Перевірити кожен файл на зміни (show diff)
            2.2 Додати *файли* до комміту
            2.3 Додати *унікальний* коментар до комміту
            2.4 Зробити комміт
            2.5 Зробити pull з origin/main (якщо є конфлікти мержимо їх)
            2.6 Зробити push в origin/branch

        3.  послідовність дій для створення пул реквеста (якщо зміни запушені)
            3.1 на gitHub знайти branch і натиснути на кнопку "New pull request"
            3.2 вибрати ревʼювера і написати Тайтл і додати опис
            3.3 переглянути зміни
            3.4 натиснути на кнопку "Create pull request"
            3.5 перевірити що ріквест є в списку ріквестів.
